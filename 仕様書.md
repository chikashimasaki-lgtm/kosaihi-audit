## 3. データ仕様

### 3.1 入力フォーマット
- 1行 = 1申請。区切りはタブ優先、なければカンマ
- 列順は固定（14列）。`COLS` 配列で定義

```javascript
const COLS = ["精算申請ID","関連決裁番号","支出日","事前承認日","精算決裁日",
  "金額税込","人数","社外人数","インボイス","店舗","申請科目",
  "社内役職","社外役職","事後理由"];
```

### 3.2 内部キーワード辞書
| 定数 | 内容 | 用途 |
|------|------|------|
| LUXE | 割烹,料亭,会員制,クラブ,ステーキ,鮨,寿司,銀座,久兵衛,高級,フレンチ,鉄板 | 高級店判定 |
| SENIOR | 役員,社長,会長,専務,常務,執行,本部長,部長 | 社内上位役職 |
| JUNIOR | 担当,社員,主任,係 | 社外下位役職 |
| GOLF | ゴルフ,カントリー,ゴルフ倶楽部,ゴルフクラブ,CC | ゴルフ費目 |
| GIFT | 手土産,菓子折,贈答,贈呈,ギフト,百貨店,中元,歳暮,贈り物 等 | 手土産費目 |
| TRAVEL | 出張,宿泊,ホテル,旅館,航空,新幹線,交通 | 旅費費目 |

## 4. ロジック仕様（内部仕様）

### 4.1 入力解析 `parseInput()`
1. textarea値を改行で分割、空行を除去
2. 各行をタブ（なければカンマ）で分割
3. COLS と対応づけてオブジェクト化（前後空白trim）

### 4.2 監査判定 `audit(r, cfg)`
処理順序：

**① 税抜計算**
```
tax = 税率/100, nd = 控除不可率/100
gross = 金額税込, ppl = 人数(最低1), ext = 社外人数
registered = インボイス列が「未」を含まない
body   = round(gross / (1 + tax))      // 本体価格
vat    = gross - body                   // 消費税額
taxExcl = registered ? body : body + round(vat * nd)  // 監査用税抜価額
per    = taxExcl / ppl                  // 1人あたり
internalOnly = (ext === 0)
```

**② 費目カテゴリ判定 `cat`**
- 店舗名がGOLF該当 → `ゴルフ`
- 店舗名がGIFT該当 → `手土産`
- 申請科目=旅費交通費 または 店舗名がTRAVEL該当 → `旅費`
- それ以外 → `飲食`

**③ 勘定科目判定 `judged` とフラグ生成**

| cat | judged決定 | フラグ |
|-----|-----------|--------|
| ゴルフ/手土産 | 社外あり→交際費 / 社内のみ→福利厚生費申請なら福利厚生費・他は交際費 | 申請≠判定で「ゴルフ接待／贈答を○○計上(隠れ交際費)」 |
| 旅費 | 社外あり→交際費 / 社内のみ→旅費交通費 | 社外あり&旅費交通費申請で「社外同伴の旅費(接待旅費の疑い)」 |
| 飲食 | 福利厚生費&社内のみ→福利厚生費 / 社内のみ→交際費 / 社外あり→per>基準で交際費・以下で会議費 | 会議費申請で交際費判定→「社内飲食費を会議費計上」or「1万円超(隠れ交際費)」/ 福利厚生費に社外混入→交際費化し「社外混入の福利厚生費(実態交際費)」 |

**④ 決裁プロセス判定**
```
noPrior   = 関連決裁番号が空            → "事前申請なし"
afterFact = days(事前承認日, 支出日) > 0 → "事後決裁(理由あり/なし)"
delay     = days(精算決裁日, 支出日)
delay > 猶予 → "決裁遅延(n日)"
```
※ `days(a,b) = round((Date(a) - Date(b)) / 86400000)`

**⑤ 不祥事シナリオ**
```
luxe        = 店舗がLUXE該当
seniorIn    = 社内役職がSENIOR該当
juniorOut   = 社外役職がJUNIOR該当
roleMismatch = seniorIn && juniorOut
luxe && roleMismatch → "相手方偽装疑い(高級店×役職不整合)"
hidden = (luxe || cat==ゴルフ) && (noPrior || afterFact)
       → "隠蔽目的の事前申請回避(...)"
```

**⑥ 格付け `grade`**
```
if ((luxe && roleMismatch) || hidden)            → "要改善"
else if フラグに 隠れ交際費/社内飲食費/福利厚生費/
        決裁遅延/事後決裁(理由なし)/事前申請なし/接待旅費 → "要改善(軽微)"
else if フラグに 事後決裁(理由あり)                → "注意"
else                                             → "適正"
```

**⑦ 税務取扱・適正根拠**
- `tax_treat` = 交際費なら「交際費等（損金不算入）」、他は「損金算入可能（交際費除外）」
- 適正時のみ `basis`（登録区分・費目・1人あたり・事前決裁・遅延なし・科目一致）を生成

### 4.3 戻り値
`audit()` は次のオブジェクトを返す：
`{r, gross, ppl, ext, registered, body, vat, taxExcl, per, judged, applied, delay, flags, grade, tax_treat, luxe, roleMismatch, basis, cat}`

### 4.4 描画 `render(cfg)`
1. 格付け件数を集計しKPIダッシュボードを描画
2. 全件を判定一覧テーブルに描画（格付けで行に色クラス付与）
3. 疑義案件（適正以外）を抽出し、監査内容確認書を生成
4. 確認書データを `CONFIRMATIONS` 配列に蓄積（出力用）

### 4.5 確認書の質問生成ロジック
検知フラグに応じて質問文を動的に組み立てる：
- 申請≠判定科目 … 費目別の科目根拠を質問
- 社内飲食費を会議費計上 … 会議の実態を質問
- 社外混入の福利厚生費 … 慰安行事該当性を質問
- 事前申請なし／事後決裁 … 事後決裁理由を質問
- 決裁遅延 … 遅延理由を質問
- 高級店×役職不整合 … 参加者の氏名・所属・役職・店舗選定理由を質問
- 該当なし … 補足説明を求める汎用質問

## 5. 出力仕様

### 5.1 結果CSV `downloadCsv()`
- ファイル名：`kosaihi_audit_results.csv`
- BOM付きUTF-8、ダブルクォート囲み
- 列：精算申請ID, 総合評価, 店舗, 税込, 監査用税抜, 1人あたり, 申請科目, 判定科目, 税務取扱, 検知フラグ

### 5.2 確認書CSV `downloadConfCsv()`
- ファイル名：`監査内容確認書.csv`
- 改行コード CRLF、質問1問=1行で展開
- 列：確認書発行日, 精算申請ID, 被監査部門, 総合評価, 指摘区分, 利用店舗, 金額税込, 1人あたり税抜, 税務上の扱い, 質問No, 質問内容, ご回答, 回答期限, ご回答者, 提示証跡

### 5.3 確認書Excel `downloadConfExcel()`
- ファイル名：`監査内容確認書.xls`
- HTMLテーブルを `application/vnd.ms-excel` で出力する方式
- 案件ごとに帳票形式（ヘッダ／所見／質問＋回答欄／証跡チェックリスト／回答者欄）

### 5.4 AIプロンプト `copyPrompt()`
- STEP1〜STEP3の監査方針・指摘レベル・出力形式を定義したプロンプト文字列 `PROMPT`
- Copilot / Gemini 等の外部生成AIへ貼り付けて定性判定に利用
- クリップボードAPI（`navigator.clipboard.writeText`）でコピー

## 8. エラーハンドリング・制約

| ケース | 挙動 |
|--------|------|
| 入力データなしで監査実行 | 「データがありません」案内を表示 |
| 結果なしでCSV出力 | アラート「先に監査を実行してください」 |
| 疑義なしで確認書出力 | アラート「疑義案件がありません」 |
| 不正な日付 | `days()` がNaNとなり該当判定はスキップ |
| 人数0 | 最低1名として扱い0除算を回避 |

### 8.1 既知の制約
- 店舗名・申請科目からの推定判定のため、表記揺れに弱い
- 経過措置率・税率は固定初期値（画面で変更可）
- ブラウザのクリップボード／ダウンロードAPIに依存
- 単一ファイル構成のため、ロジックとデータが密結合

## 9. テストパターン（サンプルデータ）

サンプルには以下の評価パターンが含まれる：
- 適正な会議費（1人あたり基準以下・事前決裁あり）
- 1万円超の隠れ交際費（会議費申請→交際費判定）
- 社内のみ飲食の会議費計上
- 福利厚生費への社外混入
- ゴルフ接待の費目分散（旅費交通費計上等）
- 手土産・贈答の会議費計上
- 未登録店（経過措置による税抜価額上乗せ）
- 事前申請なし／事後決裁（理由あり・理由なし）／決裁遅延
- 高級店×役職不整合（相手方偽装疑い）
- 高級店・ゴルフ×事前申請回避（隠蔽リスク）
